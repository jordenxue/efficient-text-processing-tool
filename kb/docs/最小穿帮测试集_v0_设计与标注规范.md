# 最小穿帮测试集 v0：设计与标注规范

## 目的

本轮只落一个可直接开跑的 v0 starter。

- `Clean Canon` 是主分数池。
- `Author-Conflict Register` 只做诊断，不进主分数。
- 本轮 starter 采用 `CC-###` / `AC-###` 编号风格。

## v0 收录边界

只收已确认、已冻结、已保持分离的项目事实。

- 可以直接作为硬事实的来源：`STATE.json`、`CURRENT_STATE.md`、`DECISIONS.md`
- 不可直接写成硬事实：relationship / location 候选联动、authoring protocol 的未来设想、未确认剧情分支、探索性备忘录

## Clean Canon

`clean_canon_testset_v0.jsonl` 进入主分数池。

每条记录字段固定为：

- `case_id`
- `mode`
- `prompt`
- `target_type`
- `canon_fact`
- `expected_behavior`
- `scoring_mode`
- `severity`
- `notes`

本轮 starter 只使用这两个 `target_type`：

- `character_fact`
- `naming_consistency`

本轮 starter 只允许：

- `mode`: `continue_writing` / `rewrite`
- `scoring_mode`: `main_score`

Clean Canon 的目标是检查模型是否破坏已确认事实，而不是让模型猜未来剧情。

## Author-Conflict Register

`author_conflict_register_v0.jsonl` 不进主分数，只做诊断登记。

每条记录字段固定为：

- `conflict_id`
- `conflict_type`
- `evidence_a`
- `evidence_b`
- `current_status`
- `policy`
- `scoring_mode`
- `notes`

本轮如果没有来自已读项目文件的明确、直接、可指认冲突证据，就保留 starter template，不为了凑数乱填。

本轮 starter 只允许：

- `current_status`: `unresolved`
- `policy`: `must_warn_and_negotiate`
- `scoring_mode`: `diagnostic_only`

## 本轮明确不做

- 不把 relationship / location 候选联动写成硬事实
- 不把 authoring protocol 的设想内容写成 canon
- 不把未确认剧情分支塞进 Clean Canon
- 不为了凑 Author-Conflict Register 条目而伪造冲突

## 一句话收口

v0 starter 的目标不是做大而全 benchmark，而是先提供一个干净、低噪声、可直接验证 continue_writing / rewrite 的最小测试集。
