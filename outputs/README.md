# outputs 目录说明

> `outputs/` 当前默认主要承接 `experimental`。
> 这里的文件不应默认被当作长期真值区或正式结构真源。

## 当前主要内容

### `outputs/generation/`

- 当前承接 round3 prefix、pilot prompts、checkpoint cards、补充卡与旧角色卡样板。
- 这些对象主要属于实验消费输入或过渡材料。

### `outputs/ab_round3/`

- 当前承接 round3 pilot 的输出与日志。
- 明确属于实验运行产物。

### `outputs/canon/` 与 `outputs/style/`

- 当前可能保留某本小说专属基础底料或处理中间产物的历史位置。
- 后续应进一步裁清其长期物理落点。
- 本轮不做搬迁，也不重跑相关流程。

## 当前使用口径

- 若任务涉及 A/B、pilot、prompt、prefix、checkpoint 或生成输出，通常先看 `outputs/`。
- 若任务涉及长期结构、正式规范或实例真值，不应把 `outputs/` 当作默认入口。
